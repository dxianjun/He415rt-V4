# HE-415RT 发射板固件 V4.0

## 项目简介

本工程是 HE-415RT 发射板固件，基于赛元 `SC92F7322` C51 平台开发，工程文件为 `demo.uvproj`。

V4.0 版本完成新版发射板适配，重点支持：

- 新版 `SC92F7322` 引脚映射
- 24C02 外部 EEPROM 存储
- P2.6 小水量/停止灯双灯三态复用
- 发射板与从模块的完整配对流程
- 从模块 MAC 读取、蓝牙配对指令生成、EEPROM 保存与上电识别

版本记录见 `demoCode版本说明.txt`。当前记录：`20260524`，发射板功能完成，待测试。

## 硬件适配

### MCU 与工程配置

- MCU：`SC92F7322`
- 工程：`demo.uvproj`
- 默认硬件版本字符串：`Rtx415_V3.0`
- UART0：`9600 8N1`
- 烧录备注：源码注释提示 `BL51 Locate -> Code Range` 设置为 `0x100`，烧录选项选择 `DISRST`

### P2.6 双灯三态

P2.6 为反向并联双灯结构，代码已封装为明确三态接口：

| 状态 | 端口配置 | 效果 |
| --- | --- | --- |
| `LED_WATER_SMALL_ON` | P2.6 推挽输出 `0` | 小水量灯亮 |
| `LED_WATER_STOP_ON` | P2.6 推挽输出 `1` | 停止灯亮 |
| `LED_WATER_BOTH_OFF` | P2.6 输入/高阻 | 两灯都灭 |

工作模式显示：

- `MODE_1`：停止灯亮
- `MODE_2`：小水量灯亮
- `MODE_3`：大水量灯亮，P2.6 双灯灭

## 24C02 EEPROM

### 器件参数

- 器件：`24C02`
- 8-bit 写地址：`0xA0`
- 8-bit 读地址：`0xA1`
- 容量：`256` 字节
- 页写大小：按 `8` 字节兼容实现
- I2C 引脚：`P0.3=SCL`、`P0.2=SDA`

### 当前数据布局

| 地址范围 | 长度 | 内容 |
| --- | ---: | --- |
| `0x00-0x01` | 2 | 已配对标记，成功后写入 `AA 55` |
| `0x02-0x07` | 6 | 预留 |
| `0x08-0x13` | 12 | 从模块 MAC，ASCII HEX，例如 `19667F4A76FC` |
| `0x14-0xFF` | 236 | 预留 |

上电后主流程会调用：

- `Ep24c02_IsReady()`：检测 24C02 是否可读
- `Ep24C02_IsMacOk()`：读取 `0x00-0x01`，判断是否为 `AA 55`

## 串口协议

### 发射板上报帧

发射板上报帧格式：

```text
B6 cmd len data... FF
```

当前主要命令：

| 命令 | 说明 | 示例 |
| --- | --- | --- |
| `0x01` | 电源/工作模式上报 | `B6 01 01 xx FF` |
| `0x02` | 喷水模式上报 | `B6 02 01 xx FF` |
| `0x03` | 电机动作上报 | `B6 03 01 xx FF` |
| `0x20` | 读 MAC 请求 | `B6 20 00 FF` |

### 主板下发帧

主板下发帧格式：

```text
A6 cmd len data... FF
```

当前主要命令：

| 命令 | 说明 |
| --- | --- |
| `0x10` | ACK |
| `0x11` | 信息命令，当前只回 ACK |
| `0x20` | 触发读 MAC 请求流程 |

### ACK 与重发

电源、水量、电机动作上报后会启动 ACK 等待：

- ACK 超时：`TIMOUT_INFO_10MS = 50`，约 `500ms`
- 超时后允许重发一次
- 收到 ACK 后清 `uc_LastCmd`

## BT ready 与 AT 指令规则

所有本机直接发送的 AT 指令都必须显式带 `\r\n\0` 结尾。

BT 初始化第一步：

```text
AT\r\n\0
```

代码路径：

- `main()` 初始化 UART 后调用 `Bt_isReady()`
- `Bt_isReady()` 发送 `cmd_bt_ready[] = "AT\r\n\0"`
- 收到 `+ok` 后，`TryParseBtReady()` 置位 `bBtReady=TRUE`
- 如果此前已有挂起的读 MAC 请求，收到 `+ok` 后会继续发送 `B6 20 00 FF`

## 配对流程

### 触发方式

在工作状态下，2 秒内按顺序完成：

```text
喷水停止 -> 小水量 -> 大水量
```

代码路径：`PairKeySeqHandle()`。

完成序列后：

1. 如果 BT 未 ready，先发送 `AT\r\n\0`，等待 `+ok`
2. BT ready 后，发射板发送读 MAC 请求帧：`B6 20 00 FF`
3. 从模块收到 `B6 20 00 FF` 后，转出 `AT+MAC?\r\n\0`
4. 从模块返回：`+MAC:19667F4A76FC`
5. 发射板解析 `+MAC:` 后 12 位十六进制 MAC
6. 发射板生成并发送配对 AT 指令：`AT+FTM=<MAC>\r\n\0`
7. 发射板把 MAC 写入 24C02 `0x08-0x13`
8. EEPROM `0x00-0x01` 写入 `AA 55` 标记配对完成

### 超时退出

发射板发出 `B6 20 00 FF` 后进入等待 `+MAC:` 状态：

- 响应超时：`BT_MAC_RSP_TIMEOUT_10MS = 200`，约 `2s`
- 2 秒内未收到合法 `+MAC:`，自动退出配对流程
- 超时后迟到的 `+MAC:` 不写入 EEPROM
- 合法 `+MAC:` 只有在 `bBtReadMacWaiting=TRUE` 时才会保存

## 工作模式与按键

工作模式定义：

| 模式 | 值 | 说明 |
| --- | ---: | --- |
| `MODE_OFF` | 0 | 关机 |
| `MODE_SLEEP` | 1 | 待机 |
| `MODE_1` | 2 | 喷水停止/停止灯 |
| `MODE_2` | 3 | 小水量 |
| `MODE_3` | 4 | 大水量 |

按键状态机支持短按、长按、持续按和释放事件。主要行为：

- 电源键短按：待机/工作切换
- 停止键：进入 `MODE_1` 并上报 `WATER_STOP`
- 小水量键：进入 `MODE_2` 并上报 `WATER_SMAL`
- 大水量键：进入 `MODE_3` 并上报 `WATER_BIG`
- 小水量/大水量长按：持续喷水及释放上报
- 方向键：左/右连续与 180 度动作上报

## 上板验证建议

1. 编译并烧录 `demo.uvproj`
2. 确认 24C02 `0xA0/0xA1` ACK 正常
3. 确认上电后发射板发送 `AT\r\n\0`
4. 模拟/实测 BT 返回 `+ok`
5. 2 秒内按 `喷水停止 -> 小水量 -> 大水量`
6. 确认发射板发送 `B6 20 00 FF`
7. 确认从模块转出 `AT+MAC?\r\n\0`
8. 返回 `+MAC:19667F4A76FC`
9. 确认发射板发送 `AT+FTM=19667F4A76FC\r\n\0`
10. 读取 24C02，确认 `0x00-0x01 = AA 55`，`0x08-0x13 = 19667F4A76FC`
11. 测试读 MAC 请求后超过 2 秒才返回 `+MAC:`，确认不会覆盖 EEPROM
12. 掉电重启，确认 `Ep24C02_IsMacOk()` 能识别已配对标记
