## 2. 结论摘要
1. 新版硬件明确使用 `SC92F7322X TSSOP-20`，当前 `IC_MODEL = SC92F7322` 与芯片型号匹配。
2. 实际贴片存储器为 `24C02`，硬件 I2C 为 `SCL=P0.3`、`SDA=P0.2`；当前代码 I2C 仍配置在 `P1.3/P1.4`，必须调整。
3. 当前代码的按键/LED 映射存在实质冲突：`P2.5` 在新版硬件是 `K3` 输入，但代码把 `P2.5` 定义成 `CTRL_LED_K2` 输出。
4. `K4=P0.4`、右转 180 为 `P0.5`，这两个点由 `SG29F7322.xls` 确认，之前“待确认”项可关闭。
5. EEPROM 已按实物确认：芯片实际为 `24C02`，8-bit 写地址为 `0xA0`，总容量 `256` 字节；规格书显示不同厂家 24C02 的页写大小存在差异，固件建议按 `8` 字节分页保守实现。

## 3. 新版 7322 引脚映射（按 SG29F7322.xls）
1. `P1.0/INT00`：`Sens`，水平开关检测，代码当前 `SENS_PIN=P10`，匹配。
2. `P1.2/RX/TCK`：串口 RX/烧录口，代码当前 UART0 RX 使用 `P12`，匹配。
3. `P1.3/TX/TDIO`：串口 TX/烧录口，代码当前 UART0 TX 使用 `P13`，匹配。
4. `P1.6`：`BT_REST`，长低复位蓝牙，代码当前 `CTRL_BT_RST=P16`，匹配。
5. `P1.7`：`Led_W`，水平/待机红灯，低电平亮，代码当前 `CTRL_LED_R=P17`，匹配。
6. `P2.7`：`LED_K4`，大水量 LED，代码当前 `CTRL_LED_K4=P27`，匹配。
7. `P2.6`：`LED_K3`，小水量/停止灯复用；低电平小水量，高电平停止灯，切输入两灯都灭。当前代码只按普通低亮 LED 使用，需要补充复用逻辑。
8. `P2.5`：`K3`，小水量按键。当前代码错误地定义为 `CTRL_LED_K2=P25`，必须修改。
9. `P2.4`：左转微调，代码当前 `IN_LEFT_ADJ=P24`，匹配。
10. `P2.1`：左转 180，代码当前 `IN_LEFT_180=P21`，匹配。
11. `P2.0`：右转微调，代码当前 `IN_RIGHT_ADJ=P20`，匹配。
12. `P0.5`：右转 180，代码当前 `IN_RIGHT_180=P05`，匹配。
13. `P0.4`：`K4`，大水量按键，代码当前 `IN_K4_PIN=P04`，匹配。
14. `P0.3`：`SCL`，24C02 I2C 时钟。当前代码冲突：`IN_K3_PIN=P03` 且 I2C 未使用此脚。
15. `P0.2`：`SDA`，24C02 I2C 数据。当前代码未作为 EEPROM SDA 使用。
16. `P0.1`：`K2`，喷水停止，代码当前 `IN_K2_PIN=P01`，匹配。
17. `P0.0`：`K1`，开关按键，代码当前 `IN_K1_PIN=P00`，匹配。

## 4. 必须改动项
1. 修正 `include/global.h` 引脚宏。
   - `IN_K3_PIN` 从 `P03` 改为 `P25`。
   - 移除或重定义 `CTRL_LED_K2=P25`，避免把 K3 输入脚配置为输出。
   - 明确 `P2.6` 的小水量/停止灯复用含义，不再按简单 `LED_MODE2_ON/OFF` 处理。

2. 修正 `user/init/IO_Init.c`。
   - `P25` 必须配置为输入上拉，而不是推挽输出。
   - `P26` 的输出/输入切换策略需按“低=小水量，高=停止灯，输入=都灭”设计。
   - `P03/P02` 必须预留给 EEPROM I2C，不能再作为按键输入使用。

3. 修正 `user/key/key.c`。
   - `InitKeyGpio()` 加入 `P25_Quasi_Mode`。
   - 移除对 `P03` 作为 K3 的依赖。
   - `GetGpioKeyIndex()` 中 K3 读取改为 `P25`。

4. 修正 I2C 驱动引脚。
   - `user/i2c/i2c_master.c`：`M_SCL` 从 `P13` 改为 `P03`，`M_SDA` 从 `P14` 改为 `P02`。
   - `Init_mI2C()` 改为配置 `P03/P02` 准双向或适合开漏模拟 I2C 的模式。
   - 若保留 `i2c2_master.c`，同步处理或明确弃用，避免维护时误用。

5. 接入 EEPROM 初始化。
   - 在主流程中加入 `Init_mI2C()`。
   - 在启用 EEPROM 读写前确认 `ep24cx_i2c.h`、`i2c_master.c`、`ep24Cx_i2c.c` 已包含到 Keil 工程编译列表。

## 5. 已确认项
1. EEPROM 设备地址。
   - 实物确认芯片为 `24C02`。
   - 结论：8-bit 写地址为 `0xA0`，8-bit 读地址为 `0xA1`。
   - 处理建议：`ADDR_EP24Cxx = 0xA0` 可以保留。

2. EEPROM 容量与页大小。
   - 实物确认：`24C02`，总容量 `256` 字节。
   - 规格书确认：常见 Microchip/Atmel/Tiger 24C02 为 `8-byte page write`；ST M24C02 系列资料为最多 `16` 字节 page write。
   - 结论：如果只按“24C02”通用兼容处理，`BYTE_PAGE` 建议改为 `8`；8 字节分页可兼容 16 字节页器件，反过来则可能跨页回卷覆盖数据。
   - 处理建议：修正注释，把当前 `24C32/64-64BYTE` 改为当前硬件实际说明；若最终确认实物完整型号支持 16 字节页，再把 `BYTE_PAGE` 调回 `16`。

3. `P2.6` 双灯复用硬件。
   - 实物确认：`P2.6` 为反向并联双灯结构，并且有上下拉电阻。
   - 原理图显示 D4 经 R5 接到 VCC，另一端接 `P2.6`；D5 从 `P2.6` 接到 R6 再到 GND。
   - 结论：`P2.6=0` 点亮 D4，`P2.6=1` 点亮 D5，`P2.6` 配成输入/高阻时两灯都灭。
   - 处理建议：新增三态控制接口，例如“小水量灯亮 / 停止灯亮 / 两灯灭”，不要再用普通 `LED_MODE2_ON/OFF` 语义。
