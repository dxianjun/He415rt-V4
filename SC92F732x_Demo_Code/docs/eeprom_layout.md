# HE-415 24C02 EEPROM 规划

## 硬件信息
实物确认存储器为 `24C02`。

关键参数：
1. 8-bit 写地址：`0xA0`
2. 8-bit 读地址：`0xA1`
3. 容量：`256` 字节
4. 内部地址宽度：`1` 字节
5. 固件页写大小：`8` 字节

说明：不同厂家的 24C02 page write 大小存在差异。常见 Microchip/Atmel/Tiger 24C02 为 8-byte page write，部分 ST M24C02 系列支持最多 16-byte page write。为兼容未知厂家丝印，固件按 `8` 字节分页拆包，避免跨页回卷覆盖数据。

## I2C 引脚
| 信号 | MCU 引脚 | 说明 |
| --- | --- | --- |
| `SCL` | `P0.3` | 24C02 时钟线 |
| `SDA` | `P0.2` | 24C02 数据线 |

## 当前业务存储布局
24C02 当前只保存“从模块”的 MAC 地址。

| 地址范围 | 长度 | 格式 | 用途 |
| --- | ---: | --- | --- |
| `0x00` - `0x0B` | 12 | ASCII HEX | 从模块 MAC，例如 `19667F4A76FC` |
| `0x0C` - `0xFF` | 244 | 保留 | 后续参数扩展 |

当前实现不额外写 Magic、版本号或校验字节，目的是让上板调试时可以直接从 `0x00` 起读到完整 MAC 文本。后续如要保存更多参数，可以从 `0x0C` 之后扩展，或升级为带版本/校验的结构。

## 配对与写入流程
1. 设备在开机/工作状态下检测按键序列：`喷水停止 -> 小水量 -> 大水量`。
2. 序列必须在 `2` 秒内完成。
3. 触发后，发射板先确认 BT ready：发送 `AT\r\n\0`，收到 `+ok` 后继续。
4. 发射板发送读 MAC 请求帧：`B6 20 00 FF`。
5. 从模块收到该请求帧后，转出：`AT+MAC?\r\n\0`，读取从机端蓝牙模块 MAC。
6. 从模块返回，例如：`+MAC:19667F4A76FC`。
7. 固件解析 `+MAC:` 后面的 12 位十六进制字符。
8. 如果发出读 MAC 请求帧后 2 秒内没有收到合法 `+MAC:` 返回，则自动退出配对流程。
9. 解析成功且仍处于等待响应状态时，将 12 字符 ASCII MAC 写入 24C02 地址 `0x00`。

## 读写策略
1. EEPROM 底层按 `8` 字节页边界拆包写入。
2. MAC 长度为 12 字节，因此会拆成 `8 + 4` 两次页写。
3. 只在返回格式满足 `+MAC:` 且后续 12 位均为十六进制字符时写入。
4. 普通 `+...` 串口返回仍会透过串口打印，但不会写 EEPROM。

## 验证项
1. 上板确认 `0xA0/0xA1` ACK。
2. 执行 `停止 -> 小水量 -> 大水量`，确认发射板串口发送 `B6 20 00 FF`。
3. 确认从模块收到后转出 `AT+MAC?\r\n\0`。
4. 从模块返回 `+MAC:19667F4A76FC` 后，读取 24C02 `0x00-0x0B`，应为 ASCII `19667F4A76FC`。
5. 测试非法返回，例如 `+MAC:XYZ` 或长度不足，不应覆盖 EEPROM。
6. 测试读 MAC 请求后超过 2 秒才返回 `+MAC:`，不应覆盖 EEPROM。
7. 掉电重启后读取 `0x00-0x0B`，确认 MAC 保持。
